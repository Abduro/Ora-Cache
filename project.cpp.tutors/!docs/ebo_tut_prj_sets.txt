::*important*:
:: in the project file each node <ItemDefinitionGroup>/<ClCompile> must contain <PreprocessorDefinitions> sub-node for working preprocessor defs properly:
:: for example:

   <ItemDefinitionGroup Condition="'$(Configuration)|$(Platform)'=='Debug|x64'">
      <ClCompile>
         <PreprocessorDefinitions>_DEBUG;WIN64;%(PreprocessorDefinitions)</PreprocessorDefinitions>
      </ClCompile>
   </ItemDefinitionGroup>

:: or Project Properties >> C/C++ >> Preprocessor >> Preprocessor Definitions:

_DEBUG;WIN32;%(PreprocessorDefinitions)   set for {Debug|x86}
_DEBUG;WIN64;%(PreprocessorDefinitions)   set for {Debug|x64}

NDEBUG;WIN32;%(PreprocessorDefinitions)   set for {Release|x86}
NDEBUG;WIN34;%(PreprocessorDefinitions)   set for {Release|x64}

::
:: The preprocessor definition must contain 'WIN64' or 'WIN32' either in accordance with project's platform (x64/Win32);
::
::*important*: in order to avoid warning LNK4003: invalid library format; library ignored ...
::
::Settings of all projects for configurations {Debug|Release} and platforms {x64|Win32} must have:

<ItemDefinitionGroup Condition="'$(Configuration)|$(Platform)'=='{Release|Debug}|{Win32|x64}'">
  <ClCompile>
    <RuntimeLibrary>MultiThreaded</RuntimeLibrary> <!--'Release'-->
<!--or-->
    <RuntimeLibrary>MultiThreadedDebug</RuntimeLibrary> <!--'Debug'-->
  </ClCompile>
</ItemDefinitionGroup>

:: or Project Properties >> C/C++ >> Code Generation >> Runtime Library:

Multi-threaded Debug (/MTd) set for {Debug} configuration
Multi-threaded (/MT)        set for {Release} configuration

:: A couple more things:
:: Project Properties >> C/C++ >> General >>
:: Debug Information Format => Program Database (/Zi) ; for all configs and platforms;
:: Support Just My Code Debugging => Yes (/JMC) ; for all configs and platforms;

:: Managing pre-compiled headers;
:: Taking into account the some projects' codebase is frequently changed, the project build efficiency should be taken into consideration;
:: The steps which are required for setting using pre-compiled headers are the following:

:: step #1: creating the pre-compiled header (*.h) file and its implementation (*.cpp);
:: these files can be placed in shared folder for easy including those files to a project configuration;

:: step #2: configuring a project file; it is necessary to specify the option of using header files:

for each configuration put the following options as in the example below:

  <ItemDefinitionGroup Condition="'$(Configuration)|$(Platform)'=='Debug|x64'">
    <ClCompile>
      <PrecompiledHeader>Use</PrecompiledHeader>                       <!--this set the option of using pre-compiled header file to 'on';-->
      <PrecompiledHeaderFile>shared.std.afx.h</PrecompiledHeaderFile>  <!--this explicitely set the pre-compiled header file name ;-->
      <ForcedIncludeFiles>shared.std.afx.h;%(ForcedIncludeFiles)</ForcedIncludeFiles> <!--this is for do *not* including healder file to source files, it will be made automatically;-->
    </ClCompile>
  </ItemDefinitionGroup>

the forcing to include pre-compiled header file can be made by project properties dialog:
C/C++ >> Advanced >> Forced Include Files => shared._std_afx.h;%(ForcedIncludeFiles) ;

and finally, that is very important, it is required to specify the implementation of the pre-compiled header files, otherwise it won't be created:

  <ItemGroup>
    <ClInclude Include="$(Shared)shared.std.afx.h" />
    <ClCompile Include="$(Shared)shared.std.afx.cpp" >
      <PrecompiledHeader Condition="'$(Configuration)|$(Platform)'=='Debug|Win32'">Create</PrecompiledHeader>
      <PrecompiledHeader Condition="'$(Configuration)|$(Platform)'=='Release|Win32'">Create</PrecompiledHeader>
      <PrecompiledHeader Condition="'$(Configuration)|$(Platform)'=='Debug|x64'">Create</PrecompiledHeader>
      <PrecompiledHeader Condition="'$(Configuration)|$(Platform)'=='Release|x64'">Create</PrecompiledHeader>
    </ClCompile>
  </ItemGroup>
